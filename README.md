## 🚀 部署步骤（5步搞定）

### 第一步：创建新仓库

1. 登录 GitHub 后，点击右上角的 `+` 号，选择 `New repository`
2. 仓库名字随便起，比如 `stock-picker-v57`
3. 选择 `Public`（公开的，免费额度更多）
4. 勾选 `Add a README file`
5. 点击最下面的 `Create repository` 按钮

### 第二步：上传代码文件

1. 进入刚创建的仓库页面
2. 点击 `Add file` 按钮，选择 `Upload files`
3. 把我给您的 `cloud_deploy` 文件夹里的**所有文件**都拖进去（包括 .github 文件夹）
   - `stock_picker_v57.py`（选股脚本）
   - `push_wechat.py`（微信推送脚本）
   - `requirements.txt`（依赖列表）
   - `.github/workflows/daily_pick.yml`（定时任务配置）
4. 拖进去后，拉到页面最下面，点击 `Commit changes` 按钮

### 第三步：配置微信推送密钥

1. 在仓库页面，点击顶部的 `Settings`（设置）
2. 左边菜单找到 `Secrets and variables`，点击展开，选择 `Actions`
3. 点击右边的 `New repository secret` 按钮
4. Name（名称）填：`SERVERCHAN_KEY` （注意全大写，下划线）
5. Secret（值）填：您刚才从Server酱复制的 `SendKey`
6. 点击 `Add secret` 按钮保存

### 第四步：测试一下能不能跑

1. 在仓库页面，点击顶部的 `Actions`（动作）
2. 如果看到提示说 Workflows 被禁用了，点击 `I understand my workflows, go ahead and enable them` 启用
3. 左边选择 `V5.7 每日选股推送`
4. 点击右边的 `Run workflow` 按钮，再点 `Run workflow` 确认
5. 等几分钟，就能看到运行结果了
6. 如果成功，您的微信会收到选股消息！

### 第五步：完成！

🎉 搞定！以后每个交易日早上9:15，您微信都会自动收到当日操盘明细！

## ❓ 常见问题

### Q1：为什么没收到微信消息？
- 检查 Server酱 的 SendKey 是不是填对了
- 检查 GitHub Secrets 里的名字是不是 `SERVERCHAN_KEY`（全大写）
- 去 GitHub Actions 页面看看运行日志，有没有报错

### Q2：每天几点推送？
- 每个交易日（周一到周五）早上 **9:15** 左右
- 有时候可能晚几分钟，正常现象

### Q3：免费额度够不够？
- GitHub Actions 每月2000分钟免费额度
- 每天跑一次，一次大概10-15分钟，一个月才300分钟左右
- 完全够用，放心用

### Q4：选股结果准不准？
- 和您本地电脑上的V5.7策略逻辑完全一样
- 只是数据来源换成了在线的，结果一致

### Q5：想改参数怎么办？
- 直接修改 `stock_picker_v57.py` 文件里的参数
- 改完提交到GitHub，下次运行就生效了
